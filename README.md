# BYOK Gemini Agent

A fully **serverless, client-side** AI assistant built with Expo (TypeScript). It
talks to the Gemini API (`gemini-2.5-flash`) directly from the device, runs its
own **autonomous agent loop** with local function calling, and creates tasks in
**Notion** as a built-in tool.

**Bring Your Own Key (BYOK):** you paste your own Gemini and Notion keys. They are
stored only in the device's hardware-backed keystore via `expo-secure-store` —
never hardcoded, never sent to any backend of ours. Every API call goes straight
from your phone to Google / Notion using your keys.

## How it works

```
You ──▶ ChatScreen ──▶ runAgentTurn() ─┐
                                        │  POST generateContent (contents + tools)
                                        ▼
                                   Gemini 2.5 Flash
                                        │
              ┌──── text reply ◀────────┤
              │                         │ functionCall: create_notion_task
              ▼                         ▼
         render bubble           fetch ─▶ Notion API (your key)
                                        │
                                        ▼  functionResponse fed back
                                   loop to Gemini ──┘
```

The loop (`src/agent/GeminiAgent.ts`):

1. The user's text is appended to a local `contents[]` history.
2. A `fetch` POST hits
   `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key=...`
   with the `contents` and the `tools` block.
3. If the model returns a `functionCall`, the UI shows a status
   (`Running tool: create_notion_task...`), the tool runs on-device (a `fetch`
   to `https://api.notion.com/v1/pages` with your Notion key), then a `model`
   turn (the `functionCall`) and a `user` turn (the `functionResponse`) are
   appended and we loop back to step 2.
4. If the model returns plain `text`, the loop ends and the reply is shown.

| File | Role |
| --- | --- |

## Setup

```bash
npm install

# Run in Expo Go / a dev build
npm start            # then press a / i, or scan the QR code
```

Open **Settings** and paste:

- **Gemini API key** — from [Google AI Studio](https://aistudio.google.com/apikey).
- **Notion API key** — create an internal integration at
  [notion.so/my-integrations](https://www.notion.so/my-integrations).
- **Notion database ID** — the 32-char id from your database URL. **Share the
  database with your integration**, and make sure its title property is named
  **`Name`** (the Notion default; change the property name in
  `GeminiAgent.ts` if yours differs).

Then go to **Chat** and try: *"Add a task to buy milk to my Notion."*

## Adding a tool

1. Add a `FunctionDeclaration` to `TOOLS` in `GeminiAgent.ts`.
2. Register an async executor in `TOOL_EXECUTORS` keyed by the tool name.

That's it — the loop discovers and dispatches it automatically.

## Builds & releases (CI)

`.github/workflows/release.yml` runs on every push to `main` (and via the Actions
tab). It:

1. Typechecks (`tsc --noEmit`).
2. **Android** — `expo prebuild` + Gradle `assembleRelease` → an installable APK
   (signed with the standard debug key from the Expo template).
3. **iOS** — `expo prebuild` + `xcodebuild archive` → an **unsigned** `.ipa`.
4. Publishes a **GitHub Release** (`v1.0.<run-number>`) with both attached.

> The iOS artifact is unsigned, so it is **not** directly installable. A real
> signed IPA needs an Apple Developer certificate + provisioning profile. To get
> one, either add fastlane/match signing to the `ios` job, or switch that job to
> [EAS Build](https://docs.expo.dev/build/introduction/) with an `EXPO_TOKEN`
> secret. The Android APK is installable as-is.

## Security notes

- Keys live only in `expo-secure-store` (Android Keystore / iOS Keychain).
- No analytics, no telemetry, no backend. All traffic is device → Google/Notion.
- `android/` and `ios/` are git-ignored and regenerated by `expo prebuild` in CI.
